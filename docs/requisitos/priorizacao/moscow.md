# MoSCoW

## Introdução

- Para priorizar os requisitos estamos utilizando a técnica de MoSCoW, que se baseia nos seguintes aspectos:
  - Must: Deve ter
  - Shoud: Deveria ter
  - Could: Pode ter
  - Won't Not: Não terá
- A prioridade é de **Won't Not** para menos importante até **Must** para mais importante.
